🎨 과학 블로그 자동화 시스템 가이드

프로그래밍 초보자도 15분 만에 따라 할 수 있는 무비용 평생 블로그 자동화 구축 안내서

🛠️

실제 화면에 표시되는 메뉴명으로 맞춰 제작했습니다.

본 가이드는 구글 클라우드, 블로거, 깃허브의 한국어 및 영어 실제 UI 명칭을 대조하며 작성되었습니다. 화면의 메뉴를 직접 보며 클릭하듯 차분히 따라오세요!

💡 알아두면 유익한 정보: 이 시스템은 100% 무료입니다! 구글 서버(GitHub Actions)가 스스로 가동되므로 전기세나 서버 운영비가 0원이며, 이미지 생성 및 AI 글쓰기도 무료 할당량 한도 내에서 영구 가동되도록 설계되었습니다.
1

구글 클라우드 콘솔 프로젝트 만들기 Google Cloud

스프레드시트 및 Blogger에 우리 AI 봇이 접근할 수 있도록 가상의 출입 프로젝트를 만드는 작업입니다.

  1. 먼저 Google Cloud Console에 본인의 구글 계정으로 로그인합니다.
  2. 화면 좌측 상단 구글 로고 우측의 [프로젝트 선택 (Select a project)] 드롭다운(혹은 기존에 열려 있는 프로젝트 이름 영역)을 클릭합니다.
  3. 나타난 팝업 창 우측 상단의 [새 프로젝트 (New Project)]를 클릭합니다.
  4. [프로젝트 이름 * (Project name *)] 입력 필드에 원하는 이름(예: Science-Blog-Bot)을 자유롭게 작성한 뒤, 하단의 파란색 [만들기 (CREATE)] 버튼을 클릭합니다.
2

필수 기능 API 활성화하기 API 라이브러리

봇이 사용할 구글 시트와 Blogger 도구를 프로젝트에 장착해야 합니다.

  1. 구글 콘솔 왼쪽 상단의 햄버거 메뉴(≡)를 누르고, [API 및 서비스 (APIs & Services)] -> [라이브러리 (Library)] 메뉴로 들어갑니다.
  2. 검색창에 Google Sheets API를 검색하여 선택한 후, 파란색 [사용 설정 (ENABLE)] 버튼을 클릭합니다.
  3. 다시 라이브러리로 돌아가 검색창에 Blogger API를 검색하여 선택한 후, 동일하게 [사용 설정 (ENABLE)] 버튼을 클릭합니다.
⚠️ 매우 중요: 두 가지 API 모두 반드시 "사용 설정" 상태여야만 봇 가동 시 에러가 나지 않습니다.
3

OAuth 동의 화면 구성하기 보안 설정

내 구글 계정 정보를 봇에게 안전하게 넘겨주기 위한 최초 동의서 설정입니다. 구글 클라우드 콘솔의 UI 버전에 따라 경로 A 또는 경로 B로 진입할 수 있습니다.

  1. [진입 경로 선택] 왼쪽 사이드바 탐색 메뉴(≡)를 누르고 다음 중 화면에 보이는 메뉴를 클릭합니다:
    • 경로 A (최신 UI): [Google 인증 플랫폼 (Google Auth Platform)]을 클릭합니다.
    • 경로 B (기존 UI): [API 및 서비스 (APIs & Services)] -> [OAuth 동의 화면 (OAuth consent screen)]을 클릭합니다.
  2. User Type (사용자 유형) 설정:
    • 경로 A (최신 UI): 상단 또는 사이드 메뉴의 [잠재고객 (Audience)] 탭을 클릭하여 User Type을 [외부 (External)]로 설정하고 [저장]합니다.
    • 경로 B (기존 UI): 화면에서 [외부 (External)] 라디오 버튼을 선택하고 하단의 [만들기 (CREATE)]를 클릭합니다.
  3. 앱 정보 등록 및 브랜드 구성 단계:
    • [앱 이름 (App name)] 입력 (예: Science Blog Bot)
    • [사용자 지원 이메일 (User support email)] 목록에서 본인 이메일 주소 선택
    • 최하단 [개발자 연락처 정보 (Developer contact information)]의 [이메일 주소 (Email addresses)]란에 본인 이메일 주소 입력
    • 맨 아래 [저장하고 계속하기 (SAVE AND CONTINUE)] 클릭 (최신 UI의 경우 우측 상단 또는 하단의 [저장 (Save)] 클릭)
  4. 범위 (Scopes) 단계: 건드릴 필요 없이 바로 최하단의 [저장하고 계속하기 (SAVE AND CONTINUE)] 클릭
  5. 테스트 사용자 (Test users) 설정 (⭐가장 중요, 누락 시 로그인 에러 발생):
    • 화면 중간의 [+ 사용자 추가 (+ ADD USERS)] 버튼을 클릭합니다.
    • 나타난 팝업창 입력란에 본인의 구글 이메일 주소(블로그 발행용 계정)를 정확히 입력하고 [추가 (ADD)]를 누릅니다.
    • 맨 아래 [저장하고 계속하기 (SAVE AND CONTINUE)] 클릭
  6. 마지막 요약 (Summary) 화면 최하단에서 [대시보드로 돌아가기 (BACK TO DASHBOARD)]를 클릭하여 동의 화면 구성을 마칩니다.
4

비밀번호 발급 (OAuth 클라이언트 ID) 인증 정보

우리 봇이 로그인 시 사용할 아이디와 비밀번호를 획득하는 과정입니다.

  1. [사용자 인증 정보 페이지 진입] 왼쪽 사이드바 탐색 메뉴(≡)를 누르고 다음 중 하나로 이동합니다:
    • 경로 A (최신 UI): [Google 인증 플랫폼 (Google Auth Platform)] -> 하위 메뉴 [클라이언트 (Clients)] 클릭
    • 경로 B (기존 UI): [API 및 서비스 (APIs & Services)] -> [사용자 인증 정보 (Credentials)] 클릭
  2. 페이지 상단(또는 중간)의 [+ 사용자 인증 정보 만들기 (+ CREATE CREDENTIALS)] 버튼을 클릭하고, 드롭다운에서 [OAuth 클라이언트 ID (OAuth client ID)]를 선택합니다.
  3. [애플리케이션 유형 (Application type)] 드롭다운 메뉴에서 반드시 데스크톱 앱 (Desktop App)을 선택합니다.
  4. [이름 (Name)] 필드는 기본값 그대로 두거나 원하는 이름을 지정한 후, 하단의 파란색 [만들기 (CREATE)] 버튼을 클릭합니다.
  5. 생성 완료 팝업창(OAuth 클라이언트 생성됨 / OAuth client created)이 표시되면, 화면에 나오는 [클라이언트 ID (Client ID)]와 [클라이언트 보안 비밀번호 (Client Secret)]를 각각 복사하여 로컬 메모장에 안전하게 보관한 후 [확인 (OK)]을 누릅니다.
5

영구 비밀번호 (Refresh Token) 획득 로컬 1회 가동

서버가 우리 구글 계정에 평생 만료 없이 로그인할 수 있도록 해주는 핵심 토큰을 로컬 컴퓨터에서 발급받는 단계입니다.

  1. 컴퓨터에 파이썬이 설치되어 있어야 합니다. (설치 시 'Add Python to PATH' 체크 필수)
  2. 명령 프롬프트(윈도우 cmd) 또는 터미널을 열고 아래 명령어로 필요한 패키지를 설치합니다: 
    pip install google-auth-oauthlib google-api-python-client pillow requests
  3. 프로젝트 경로 내의 src 폴더로 터미널 경로를 이동하거나, get_refresh_token.py를 로컬에서 실행합니다: 
    python src/get_refresh_token.py
  4. 콘솔창 안내에 따라 4단계에서 획득한 Client ID와 Client Secret을 차례로 붙여넣고 엔터를 칩니다.
  5. 자동으로 인터넷 브라우저 창이 뜨며 구글 로그인 화면이 열립니다. **3단계에 기재했던 내 테스트 계정 구글 이메일**로 로그인합니다.
  6. '안전하지 않은 앱' 경고창이 뜰 경우, 당황하지 마시고 고급 > (안전하지 않음)으로 이동을 눌러 승인해 주시면 됩니다.
  7. 모든 권한 동의를 마치면 터미널 창에 GOOGLE_REFRESH_TOKEN 문자열이 나타납니다. 전체를 드래그하여 안전한 메모장에 보관하십시오!
6

스프레드시트 및 Blogger ID 확보 블로그/시트

실제 정보들이 쓰이고 발행될 영역들의 고유 주소를 확보합니다.

  • 구글 스프레드시트 ID (Spreadsheet ID): 새 시트를 만들고 주소창을 봅니다.
    https://docs.google.com/spreadsheets/d/1aBcDeFgHiJkLmNoP.../edit
    중간에 존재하는 대소문자와 숫자, 기호 조합이 바로 스프레드시트 ID입니다.
  • Blogger 블로그 ID (Blog ID): Blogger 관리자 화면에는 별도의 블로그 ID 항목이 텍스트 형태로 쉽게 드러나지 않아 헷갈릴 수 있습니다. 가장 정확하고 빠른 방법은 본인 Blogger 화면의 **웹 브라우저 주소창(URL)**을 확인하는 것입니다.
    https://www.blogger.com/blog/posts/847596827... 또는 https://www.blogger.com/blog/settings/847596827...
    주소창에서 posts/ 혹은 settings/ 바로 뒤에 나타나는 19자리의 고유 숫자 조합이 바로 본인의 블로그 ID입니다.
7

로컬 코드를 GitHub 저장소에 올리기 GitHub 연동

내 컴퓨터의 블로그 시스템 소스 코드를 본인의 원격 깃허브 저장소로 업로드하고 연동해야 크론 스케줄러가 작동합니다.

  1. 본인의 GitHub 계정에 로그인한 뒤, 우측 상단 [+] 아이콘 -> [New repository] 버튼을 누릅니다.
  2. 저장소 이름(Repository name)을 입력합니다. (예: science-blog-bot)
  3. ⚠️ **아주 중요:** 저장소 공개 범위는 반드시 Public으로 설정해야만 무료 이미지 CDN 기능(GitHub Pages)을 연동할 수 있습니다.
  4. 다른 초기화 옵션(README.md 추가 등)은 체크하지 않고 최하단의 [Create repository]를 클릭합니다.
  5. 프로젝트 경로인 C:\Users\jacho\Documents\antigravity\lucid-hawking 에서 명령 프롬프트(cmd) 또는 터미널을 열고 다음의 연동 명령어를 한 줄씩 차례대로 엔터를 치며 입력합니다 (원격 주소는 본인 깃허브 아이디로 수정하십시오):
git init
git add .
git commit -m "Initial commit of blog bot"
git branch -M main
git remote add origin https://github.com/본인의_깃허브_아이디/science-blog-bot.git
git push -u origin main

※ 최초 푸시 시 구글/깃허브 로그인 창이 떠서 본인 인증을 요구할 수 있습니다. 정상적으로 로그인을 완료하면 업로드가 성공합니다.

8

GitHub Secrets에 비밀번호 등록하기 Actions 세팅

우리 봇이 있는 GitHub 저장소에 안전하게 모든 API 비밀키들을 매핑합니다. 외부 유출 위험이 전혀 없는 보안 금고입니다.

  1. 본인의 GitHub 저장소 상단 탭 중 톱니바퀴 아이콘인 [Settings] 메뉴를 클릭합니다.
  2. 좌측 사이드바 메뉴 아래 [Security] 대메뉴 하위에서 [Secrets and variables]를 찾고, 화살표를 눌러 하위 메뉴의 [Actions]를 클릭합니다.
  3. Repository secrets 섹션 우측의 초록색 [New repository secret] 버튼을 클릭합니다.
  4. 아래 6가지 정보를 [Name]에 정확히 적고, [Secret] 입력란(최신 GitHub UI에서는 Value가 아닌 Secret으로 명시됨)에 값을 붙여넣은 뒤 [Add secret]을 클릭해 하나씩 등록합니다.
Name (Secret 이름, 정확히 입력) Secret (입력할 실제 비밀값)
GEMINI_API_KEY Google AI Studio에서 발급받은 무료 API 키
GOOGLE_SHEETS_ID 6단계에서 획득한 스프레드시트 고유 ID
BLOG_ID 6단계에서 획득한 Blogger 숫자 ID
GOOGLE_CLIENT_ID 4단계에서 획득한 OAuth 클라이언트 ID
GOOGLE_CLIENT_SECRET 4단계에서 획득한 OAuth 보안 비밀번호
GOOGLE_REFRESH_TOKEN 5단계에서 터미널을 실행하여 얻어낸 만료 없는 Refresh Token
🔑 **추가 팁 (Workflow 쓰기 권한 활성화):**
가상 머신이 이미지 브랜치에 직접 글을 업로드할 수 있도록 권한을 수동 활성화해야 합니다.
  1. 저장소 상단 [Settings] -> 좌측 [Actions] -> [General] 메뉴로 이동합니다.
  2. 맨 하단으로 스크롤하여 [Workflow permissions] 섹션을 찾습니다.
  3. [Read and write permissions] 라디오 버튼을 선택하여 활성화하고, 바로 아래의 [Save] 버튼을 반드시 클릭하여 완료합니다.
9

GitHub Pages 이미지 호스팅 세팅 무료 호스팅

자동 생성된 이미지를 구글 Blogger 본문에 노출하기 위한 초고속 무료 이미지 CDN 주소를 얻는 세팅입니다.

  1. GitHub 저장소 [Settings] -> 좌측 [Pages] 탭으로 이동합니다.
  2. 중간 Build and deployment 섹션의 Source 드롭다운을 [Deploy from a branch]로 둡니다.
  3. ⚠️ **주의:** 최초 1회 봇이 실행된 이후라야 브랜치 선택 란에 images-hosting 이라는 브랜치가 활성화됩니다. 10단계 실행 후 이 메뉴로 다시 돌아와 브랜치 드롭다운에서 [images-hosting]으로 변경하고, 폴더는 [/ (root)]를 선택한 뒤 바로 옆의 [Save] 버튼을 꼭 클릭해 주십시오.
  4. 활성화가 완료되면 이미지가 업로드될 때마다 https://본인아이디.github.io/저장소이름/ 주소로 웹에 무료 무제한 퍼블릭 CDN이 열려 포스트 본문에 자동 바인딩됩니다.
10

동작 시키고 확인하기 수동 드라이런

이제 모든 세팅이 완료되었습니다. 6시간 간격으로 저절로 돌지만, 즉시 작동하는지 테스트해봅니다.

  1. GitHub 저장소의 상단 탭에서 [Actions] 탭을 클릭합니다.
  2. 좌측의 **Workflows** 목록에서 Premium Science Blog Auto-Publisher 를 선택합니다.
  3. 우측의 회색 [Run workflow] 드롭다운 메뉴를 클릭하고, 나타난 팝업창에서 초록색 [Run workflow] 버튼을 다시 한번 클릭합니다.
  4. 잠시 후 노란색 진행 회전 아이콘이 돌며 파이썬 파이프라인이 실행됩니다. (완료까지 약 1~2분 소요)
  5. 완료 시 구글 스프레드시트에 자동으로 '설정' 탭과 '로그' 탭이 생성되면서, 오늘 분석하여 발행한 글의 제목과 Blogger URL 링크가 실시간 누적 기록된 것을 확인하실 수 있습니다!
